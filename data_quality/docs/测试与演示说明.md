# 测试与演示说明

## 1. 演示接口

本地演示接口用于汇报展示规则库和智能推荐能力，不依赖完整 OpenMetadata 环境。

实现文件：

```text
data_quality/demo/quality_rule_demo_server.py
```

启动命令：

```powershell
python data_quality/demo/quality_rule_demo_server.py --port 8765
```

访问页面：

```text
http://127.0.0.1:8765
```

页面支持：

- 查看规则库概览
- 点击执行测试
- 查看测试结果
- 查看规则库列表
- 查看智能推荐结果
- 查看手机号 SQL 脚本预览

## 2. 演示接口清单

| 接口 | 说明 |
|---|---|
| `/` | 汇报演示页面 |
| `/api/summary` | 规则库概览 |
| `/api/rules` | 规则列表 |
| `/api/recommendations` | 智能推荐结果 |
| `/api/preview` | 规则脚本预览 |
| `/api/report` | 静态测试报告 |
| `/api/run-test` | 点击后实时执行测试 |

流程冒烟测试脚本：

```powershell
python data_quality/demo/quality_workflow_smoke_test.py
```

## 3. 测试接口说明

`/api/run-test` 会实时执行以下检查：

| 检查项 | 内容 |
|---|---|
| 规则库内置规则数量 | 校验是否为 95 条 |
| 六维评价规则覆盖 | 校验六大维度是否齐全 |
| SQL/GE/ETL 覆盖 | 校验三种执行方式是否均覆盖 |
| 手机号规则脚本预览 | 校验 SQL 预览是否能正确渲染 |
| 智能推荐关键规则命中 | 校验手机号、身份证、金额、枚举、关联、跨系统规则是否命中 |
| 智能推荐结果数量 | 校验推荐结果数量是否符合预期 |
| 推荐结果脚本预览 | 校验推荐结果中是否包含脚本预览 |

成功返回示例：

```json
{
  "test_result": "PASS",
  "total_checks": 7,
  "passed_checks": 7,
  "failed_checks": 0
}
```

## 6. 注意事项

当前演示服务通过最小依赖桩加载模块，主要用于汇报演示。

如果要接入真实平台，需要：

- 安装完整 `openmetadata-ingestion` 依赖
- 接入真实元数据采集结果
- 将 API 门面对接 FastAPI 或现有后端路由
- 将确认后的规则写入正式数据库或 OpenMetadata TestCase
