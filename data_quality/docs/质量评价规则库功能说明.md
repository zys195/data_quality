# 质量评价规则库功能说明

## 1. 建设目标

质量评价规则库用于统一管理标准规则、业务规则、技术规则和自定义规则，为质量评价任务提供可复用、可统计、可预览、可落地的规则模板。

规则库面向白皮书中提出的质量检核能力，覆盖：

- 规则模板管理
- 规则查询与统计
- 检核任务规则复用
- 问题分类与整改建议
- SQL、GE、ETL 多方式落地

## 2. 实现位置

| 能力 | 文件 |
|---|---|
| 规则库核心模型与服务 | `data_quality/rules/rule_library.py` |
| 规则库 API 请求/响应模型 | `data_quality/api/rule_library_api.py` |
| 包导出 | `data_quality/rules/__init__.py`、`data_quality/api/__init__.py`、`data_quality/__init__.py` |

## 4. 规则模板字段

每条规则统一包含以下字段：

| 字段 | 说明 |
|---|---|
| `rule_id` | 规则编号 |
| `name` | 规则英文名称 |
| `display_name` | 规则展示名称 |
| `dimension` | 评价维度 |
| `source_type` | 规则来源：标准、业务、技术、自定义 |
| `problem_category` | 问题归类 |
| `core_definition` | 核心定义 |
| `applicability` | 适用对象、数据类型、字段名模式、业务标签 |
| `scripts` | SQL、GE、ETL 执行脚本 |
| `test_definition_name` | 对应测试定义名称 |
| `parameters` | 可调规则参数 |
| `threshold` | 阈值配置 |
| `validation_level` | 强弱校验级别 |
| `severity` | 严重程度 |
| `responsible_role` | 责任角色 |
| `remediation_suggestion` | 整改建议 |
| `issue_strategy` | 问题处理策略 |
| `tags` | 规则标签 |
| `status` | 规则状态 |
| `version` | 版本 |

## 5. 六维规则覆盖

当前内置规则共 95 条，其中原有 35 条基础规则继续保留，并按白皮书六维评价体系新增 60 条扩展规则。

| 维度 | 数量 | 示例 |
|---|---:|---|
| 规范性 | 22 | 身份证、手机号、车牌号、IMEI、MAC、IP、URL、邮箱、编码、行政区划、命名、枚举值、日期、时间戳、信用代码、银行卡、币种、表字段命名等 |
| 完整性 | 16 | 必填字段、字段填充率、空白字符串、主键唯一性、记录完整性、关联完整性、组合完整性、分区完整性、成对字段完整性等 |
| 准确性 | 17 | 数值型、取值范围、金额精度、长度、多字段条件、时间先后逻辑、未来时间、金额非负、身份证生日/性别、税额计算、脏值等 |
| 一致性 | 13 | 跨系统一致性、计算逻辑一致性、主从表核心字段一致性、码值字典、事实维表、组织层级、单位换算、血缘转换等 |
| 时效性 | 13 | 数据新鲜度、ETL 加工延迟、批次完成时效、源端同步、分区到达、报表发布、API缓存、增量水位、元数据更新等 |
| 可访问性 | 14 | 表可访问性、访问权限、查询响应时效、API 接口可用性、服务SLA、脱敏、目录检索、审计、备份恢复等 |

## 7. API 能力

规则库 API 位于 `RuleLibraryAPI`。

| 方法 | 说明 |
|---|---|
| `import_rules()` | 导入规则 |
| `create_rule()` | 新增规则 |
| `update_rule()` | 修改规则 |
| `delete_rule()` | 删除规则 |
| `get_rule()` | 查询规则详情 |
| `query_rules()` | 查询规则列表 |
| `preview_rule_script()` | 规则脚本预览 |
| `reuse_rule()` | 规则复用 |
| `get_summary()` | 规则库统计 |
